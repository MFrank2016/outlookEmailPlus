# 微软云配置自动化提示词

> 用途：把 **Azure / Microsoft Entra 云端配置** 这部分交给其他 AI 或自动化代理执行。
>
> 范围：**仅处理微软云侧配置**，不处理本地项目代码、浏览器授权、token 导入、账号写入。

---

## 1. 可直接发给其他 AI 的详细提示词

```markdown
你是一个负责 **Microsoft Entra / Azure App Registration 自动化配置** 的工程代理。

你的目标不是修改本地项目代码，而是**只完成微软云端配置**，让一个 Outlook / Microsoft personal account 兼容导入链路满足以下约束：

## 目标

请使用 **Azure CLI / Microsoft Graph API / Microsoft Graph PowerShell**（任选其一或组合）完成或校验以下配置：

1. **Supported account types / audience**
   - 最终必须是：
     - `AzureADandPersonalMicrosoftAccount`
   - 不要保留：
     - `AzureADMyOrg`
     - `PersonalMicrosoftAccount`

2. **Public Client**
   - 必须开启 public client / fallback public client
   - 等价目标：
     - `allowPublicClient = true`
     - 或 Graph 新模型中的 `isFallbackPublicClient = true`

3. **Redirect URI / Platform**
   - 必须配置 **Mobile and desktop applications** 平台
   - 必须包含以下 redirect URI：
     - `http://localhost:5000/token-tool/callback`
   - 如当前平台仍只保留 Web 回调，请补 public client redirect

4. **Token version**
   - 若修改 audience 时遇到 token version / requestedAccessTokenVersion 相关报错：
     - 需要把 token version 相关字段调到 `2`
   - 兼容目标：
     - `accessTokenAcceptedVersion = 2`
     - 或等价的 `requestedAccessTokenVersion = 2`

5. **API permissions**
   - 至少补以下 delegated permissions：
     - **Office 365 Exchange Online → IMAP.AccessAsUser.All**
     - **Microsoft Graph → Mail.Read**

## 重要边界

### 只做这些
- 读取并更新 App Registration
- 配 audience
- 配 public client
- 配 redirect URI
- 配 API permissions
- 做变更前后差异检查
- 输出最终配置摘要

### 不要做这些
- 不要尝试获取用户的 refresh_token
- 不要自动化 personal Microsoft account 的最终 consent 点击
- 不要修改本地代码仓库
- 不要生成 client_secret 并要求本地链路使用它

## 已知兼容结论

这是一个 **兼容导入模式** 场景，云端配置必须满足：

- audience 不能太窄：
  - `PersonalMicrosoftAccount` 会导致后续 `/common` 链路出现 `AADSTS9002331`
- 不能只面向组织目录：
  - 否则会出现 `unauthorized_client` / `not enabled for consumers`
- 不能继续沿 Web 机密客户端思路：
  - 否则 Azure 会要求 `client_secret`
- IMAP 权限必须放开：
  - 否则会出现 `AADSTS70000` / scope unauthorized

## 优先采用的自动化方式

### 方案 A：Azure CLI

优先尝试：

- `az ad app show`
- `az ad app update`
- `az ad app permission add`
- `az ad app permission list`
- `az ad app permission admin-consent`（如果当前环境适用）

### 方案 B：Microsoft Graph API / az rest

如果 Azure CLI 不能直接覆盖某些字段，请改用：

- `PATCH /applications/{applicationObjectId}`

重点字段：

- `signInAudience`
- `isFallbackPublicClient`
- `publicClient.redirectUris`
- `requiredResourceAccess`

### 方案 C：Microsoft Graph PowerShell

如果 PowerShell 更方便，可使用：

- `Update-MgApplication`

## 已知资源与权限（执行前请再次校验）

### 资源 AppId
- Microsoft Graph:
  - `00000003-0000-0000-c000-000000000000`
- Office 365 Exchange Online:
  - `00000002-0000-0ff1-ce00-000000000000`

### 常用 delegated permission IDs
- Microsoft Graph `Mail.Read`:
  - `570282fd-fa5c-430d-a7fd-fc8dc98a9dca`
- Exchange Online `IMAP.AccessAsUser.All`:
  - `b1d34000-793b-4a50-b2b7-2a2e1e1d4b74`

> 如果执行环境里能直接查询到最新权限元数据，请以实时查询结果为准。

## 执行要求

1. 先读取当前 App Registration 状态
2. 明确输出“当前值 → 目标值”的 diff
3. 再执行更新
4. 更新后重新读取并验证：
   - audience
   - public client
   - redirect URI
   - API permissions
5. 最终输出：
   - 实际执行的 CLI / API 调用
   - 成功修改了哪些字段
   - 哪些步骤因为权限不足或 tenant 限制未能完成

## 如果遇到阻塞，按这个顺序处理

1. **权限不足**
   - 明确指出需要的 Entra / App Registration 管理权限
   - 不要伪造成功

2. **字段在 CLI 里不好改**
   - 改用 Microsoft Graph PATCH

3. **无法自动完成 personal account consent**
   - 明确说明：
     - 云配置已完成
     - 剩余必须由用户在浏览器中手动登录 / consent

## 最终交付格式

请按以下结构输出：

1. 当前配置检查结果
2. 执行的自动化修改
3. 修改后的最终状态
4. 仍需人工完成的步骤（如果有）
5. 风险与注意事项
```

## 2. 这份提示词适合的使用方式

适合让其他 AI 代理完成：

- App Registration 新建
- audience 修正
- public client 配置
- redirect URI 修正
- API permissions 增加
- 变更前后状态检查

不适合直接声称“全自动完成”的部分：

- 用户 personal account 的最终浏览器登录
- delegated consent 的最终点击
- refresh_token 的真实获取

---

## 3. 建议你给其他 AI 时顺带附上的上下文

为了让别的 AI 真正执行成功，建议你同时提供：

1. App Registration 的 `appId` 或对象 ID
2. 当前 tenant / subscription 登录上下文
3. 是否允许它使用：
   - Azure CLI
   - Microsoft Graph PowerShell
   - `az rest`
4. 目标 redirect URI
5. 是否只做“修现有应用”，还是允许“新建一个兼容应用”

---

## 4. 最现实的落地边界

这份提示词最适合的定位不是：

- “让 AI 完整替用户拿到 refresh_token”

而是：

- **让 AI 把微软云端配置全部自动收好**
- **最后只把用户保留在浏览器里做一次人工授权**

这是当前 personal Microsoft account + delegated permissions 场景下最现实、最稳的自动化边界。
