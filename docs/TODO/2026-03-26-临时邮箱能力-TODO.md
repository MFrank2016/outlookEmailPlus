## 1. 文档目标

本文档不再讨论需求本身，而是把“临时邮箱能力”后续开发整理成可执行的 TODO 模型。

目标只有三个：

1. 让后续开发知道先做什么、后做什么。
2. 让 AI / 开发在推进时有统一任务格式。
3. 让测试、实现、联调、收尾都能挂到明确任务上。

## 2. TODO 模型定义

每个 TODO 统一按以下字段描述：

| 字段 | 说明 |
|------|------|
| `ID` | 唯一任务编号 |
| `模块` | 前端 / 后端 / 数据层 / 测试 / 文档 |
| `级别` | `P0` / `P1` / `P2` |
| `状态` | `todo` / `doing` / `blocked` / `done` |
| `任务` | 要做什么 |
| `原因` | 为什么必须做 |
| `前置` | 依赖哪些任务先完成 |
| `产出` | 代码、接口、测试或文档结果 |
| `验收` | 判定任务完成的标准 |
