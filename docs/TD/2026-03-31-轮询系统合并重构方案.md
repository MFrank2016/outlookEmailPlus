# 轮询系统合并重构方案 — 以简洁模式引擎为主

> 编写时间：2026-03-31  
> 前置文档：`docs/BUG/2026-03-30-简洁模式自动轮询-会话BUG记录.md`
> **状态：✅ 已完成（2026-03-31）**
> **测试结果：67/67 全绿**

---

## 1. 问题陈述

当前存在两套独立的轮询系统：
- **标准模式轮询**（`main.js`，~280 行）：选择账号时自动启动，单账号轮询
- **简洁模式轮询**（`mailbox_compact.js`，~400 行）：复制邮箱时启动，多账号并行

两套系统功能重叠度高（设置结构、错误处理、API 调用），但设置面板分离、代码各自维护，
导致 Bug 难以统一修复（BUG-4/BUG-6）、设置面板混乱（用户困惑"两套轮询开关"）。

**目标**：以简洁模式引擎为主体，将标准模式的轮询功能整合进来，形成**一套统一的轮询引擎**。

---

## 2. 核心差异分析

| 维度 | 标准模式 | 简洁模式 | 合并后方案 |
|------|---------|---------|-----------|
| 触发方式 | 选中账号时自动启动 | 复制邮箱时启动 | **复制邮箱时启动**（两种模式统一行为） |
| 并发 | 单账号 | 多账号并行（Map） | **多账号并行**（Map，沿用简洁模式） |
| API 调用 | 单 folder | inbox + sentitems 并行 | **inbox + sentitems 并行**（更全面） |
| 停止条件 | 次数上限 / 3次错误 | 次数上限 / 3次错误 / 发现新邮件 | **次数上限 / 3次错误 / 发现新邮件** |
| 新邮件处理 | 通知 + 红点 + 更新列表 | 提取验证码 + 复制 | **提取验证码 + 复制**（优先），失败回退到通知 |
| baseline | `knownEmailIds`（来自当前已加载邮件） | `state.baselineIds`（启动时 fetch 构建） | **启动时 fetch 构建 baseline**（更可靠） |
| UI 指示 | 邮箱地址旁小绿点 | 拉取按钮变绿 + 停止监听文字 | **根据 mailboxViewMode 分发**（见下方架构图） |
| 设置 | 独立面板：开关+间隔(5-300s)+次数 | 独立面板：开关+间隔(3-60s)+次数 | **合并为一套设置**：开关+间隔(3-300s)+次数 |
| 可见性处理 | 无 | 后台暂停/前台恢复 | **保留后台暂停/前台恢复** |

## 3. 重构方案

### 3.1 架构设计（实际实现）

```
┌─────────────────────────────────────────────────┐
│         统一轮询引擎 (poll-engine.js, 342行)      │
│                                                 │
│  ┌─ pollMap (Map<email, PollState>)            │
│  ├─ startPoll(email, opts)                     │
│  ├─ stopPoll(email, toastMsg, toastType)       │
│  ├─ stopAllPolls()                             │
│  ├─ pollSingleEmail(email, state)              │ ← 核心 poll 逻辑
│  ├─ applyPollSettings(settings)                │
│  ├─ applyPollSettingsToRunning(newSettings)     │
│  ├─ reapplyAllPollUI()                         │
│  ├─ registerPollUICallbacks(callbacks)          │ ← UI 回调注册
│  ├─ visibilitychange 监听                       │
│  └─ pollT() / _handlePollError() / ...         │
│                                                 │
│  PollState = {                                  │
│    timer, startTime, baselineIds,               │
│    errorCount, pollCount, isPolling,            │
│    intervalSec, maxCount                        │
│  }                                              │
│                                                 │
│  全局设置：pollEnabled / pollInterval / pollMaxCount │
└──────────────────────┬──────────────────────────┘
                       │
         registerPollUICallbacks()
                       │
    ┌──────────────────┴──────────────────┐
    │  mailbox_compact.js UI 回调分发层    │
    │  (根据 mailboxViewMode 自动路由)      │
    ├──────────────────────────────────────┤
    │ onPollStart(email, maxCount):        │
    │   compact → updateCompactPollUI()    │
    │   standard → showStandardPollDot()   │
    │                                      │
    │ onPollStop(email):                   │
    │   compact → updateCompactPollUI()    │
    │   standard → hideStandardPollDot()   │
    │                                      │
    │ onPollTick(email, remaining):        │
    │   compact → updateCompactPollUI()    │
    │   standard → (无需倒计时文字)         │
    │                                      │
    │ onNewEmail(email, summary):          │
    │   compact → updateSingleRowFromCache │
    │   standard → (引擎自动处理)           │
    │                                      │
    │ onAccountCheck(email):               │
    │   compact → findCompactAccountRow()  │
    │   standard → return true             │
    └──────────────────────────────────────┘
```

### 3.2 设置合并（实际实现）

**before（2 个面板 + 6 个后端字段）**：
```
标准轮询：enable_auto_polling / polling_interval / polling_count
简洁轮询：enable_compact_auto_poll / compact_poll_interval / compact_poll_max_count
```

**after（1 个面板 + 3 个活跃字段 + 3 个 deprecated 字段）**：
```
统一轮询（活跃）：enable_auto_polling / polling_interval / polling_count
已废弃（兼容）：enable_compact_auto_poll / compact_poll_interval / compact_poll_max_count
```

- `enable_auto_polling`：总开关，同时控制标准和简洁模式
- `polling_interval`：轮询间隔（秒），范围 3-300
- `polling_count`：最多轮询次数（0=持续），范围 0-100
- 简洁模式 3 个专属字段在后端标记 `[Phase 3 deprecated]`，保留读写（向后兼容），前端保存时镜像标准字段值

### 3.3 数据流（设置 → 引擎）

```
saveSettings() → applyPollingSettings(settings) → applyPollSettings({enabled, interval, maxCount})
initPollingSettings() → applyPollingSettings(data.settings) → applyPollSettings({enabled, interval, maxCount})
loadSettings() → applyPollingSettings(data.settings) → applyPollSettings({enabled, interval, maxCount})
```

### 3.4 触发时机统一（实际实现）

| 场景 | 行为 |
|------|------|
| 标准模式 - 选中账号 | **不自动启动轮询**（已移除 `syncPollingForCurrentAccount`） |
| 标准模式 - 复制邮箱 | `email-copied` → `startPoll(email)` |
| 简洁模式 - 复制邮箱 | `email-copied` → `startPoll(email)` |
| 切换到临时邮箱组 | `stopAllPolls()` |
| 账号列表重新渲染 | `reapplyAllPollUI()` |

> **行为变更**：标准模式原来是"选中就轮询"，合并后改为"复制才轮询"。
> 用户已确认此变更可接受。

## 4. 实施步骤（含实际执行结果）

### Phase 1：提取统一引擎 ✅
1. 新建 `static/js/features/poll-engine.js`（342 行）
2. 将 `mailbox_compact.js` 中的轮询引擎代码迁移至此（保留 compact-specific UI 回调注入点）
3. `mailbox_compact.js` 改为调用统一引擎，注册 UI 回调（含 `mailboxViewMode` 分发逻辑）
4. `templates/partials/scripts.html` 在 `mailbox_compact.js` 之前引入 `poll-engine.js`

### Phase 2：标准模式接入统一引擎 ✅
1. 删除 `main.js` 中 ~280 行旧轮询代码（12 个函数 + 6 个变量）
2. 添加 `showStandardPollDot`/`hideStandardPollDot` 标准模式 UI 函数
3. 添加 `applyPollingSettings` 简化版（更新内部变量 + 调用引擎 `applyPollSettings`）
4. 更新 4 个外部调用者（`accounts.js`、`emails.js`、`groups.js`、`temp_emails.js`）

### Phase 3：设置面板合并 ✅
1. `index.html`：删除简洁模式独立面板（27 行），标准轮询间隔范围从 5-300 调整为 3-300
2. `main.js`：删除 3 个 compact 全局变量，清理 `loadSettings`/`saveSettings`/`initPollingSettings`
3. `settings.py`：compact 字段标记 `[Phase 3 deprecated]`，标准间隔验证范围 5→3

### Phase 4：测试更新 ✅
1. TC-B：更新 3 个测试（compact 面板不存在 → 标准面板存在，变量不存在，applyCompactPollSettings 不在 main.js）
2. TC-A：后端测试因保留 backward compat 自然通过（12/12）
3. TC-C：引擎逻辑未变，Jest 测试自然通过（23/23）
